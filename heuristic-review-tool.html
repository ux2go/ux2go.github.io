The UX2GO Heuristic Review Tool

Our tool consists of 10 heuristics and a form of PCA (Perception-Cognition-Action) analysis that allows us to outline root causes and detail the effects along with the expected frequency and severity of each predicted issue. To achieve the best results, we recommend that the review is carried out on specific user journeys. However, we can also review individual pages.

The Heuristics

Originally based on Jakob Nielsen's 10 Heuristics (1994) and several other studies, UX2GO has refined and optimised a set of heuristics for the 21st century:

        • Consistency
        • Feedback
        • Predictability
        • Affordance
        • Familiarity
        • Flexibility
        • Tolerance
        • Simplicity
        • Accessibility
        • Aesthetic

PCA analysis

All usability issues that users experience can be attributed to a failure of the design to allow the user to use a website or app successfully. These failures stem from at least one of 3 parts of any user's interactions:

      1. Perception - The design must first convey successfully to the user that there is the option to interact. For example, a user needs to be able to see a link in order to start interacting with it.
      2. Cognition - The design must appropriately inform the users that they can understand what the interaction will do. For example, if a user sees a link but cannot work out what it will do then cognition is the point of failure.
      3. Action - The design must successfully allow the user to physically carry out the intended action. For example, a user may see a slider on a horizontal axis, understand that they can slide it back and forth, but find that it is difficult to find the right part of the slider to click because the catchment area is too small. This would make 'action' the point of failure.
